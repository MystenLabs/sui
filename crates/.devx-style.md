# Rust Code Ordering

Order items in each file as follows:

1. Imports (`use` statements)
   1. `std`
   2. External crates (third-party dependencies)
   3. `crate::` (current crate)
   4. `super::` / `self::`
2. Modules (`mod` declarations)
3. Constants and statics (`const`, `static`)
4. Type definitions (`struct`, `enum`, `union`)
5. Type aliases (`type`)
6. Inherent `impl` blocks
7. Trait implementations (`impl Trait for Type`)
8. Free functions (`fn`)
9. Tests (`#[cfg(test)]` module, follow same overall order with additional subrules)
    1. test helpers
    2. `#[test]`/ `#[tokio::test]` functions

# Rust Import Rules

1. Within each group, sort alphabetically.
2. Prefer explicit imports over globs (`use std::collections::HashMap` not `use std::collections::*`).
3. Flatten imports to one item per use statement. Do not use nested braces.
    ```rust
    // Good
    use std::collections::HashMap;
    use std::collections::HashSet;

    // Bad
    use std::collections::{HashMap, HashSet};
    ```
