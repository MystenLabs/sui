This guide walks through the complete process of setting up a Sui validator, from generating keys to registering as a validator candidate. For hardware requirements and ongoing operations, see [Validator Configuration](/operators/validator/validator-config) and [Validator Tasks](/operators/validator/validator-tasks).

## Prerequisites

- A machine meeting the [hardware requirements](/operators/validator/validator-config#hardware-requirements) for running a Sui validator
- Sui CLI installed and configured (see [Install Sui](/getting-started/onboarding/sui-install))
- Familiarity with Sui addresses and key management (see [Keys and Addresses](/develop/transactions/transaction-auth/keys-addresses))

## Step 1: Generate validator keys

A Sui validator uses four distinct key pairs, each serving a different purpose:

| Key | Purpose | Algorithm |
|---|---|---|
| **Authority key** | Signs consensus votes and certificates | BLS12-381 |
| **Protocol key** | Used for the validator's onchain identity | Ed25519 |
| **Worker key** | Authenticates worker-to-worker communication | Ed25519 |
| **Network key** | Authenticates peer-to-peer network connections | Ed25519 |

Generate all four keys using the Sui CLI:

```bash
sui keytool generate bls12381
sui keytool generate ed25519  # protocol key
sui keytool generate ed25519  # worker key
sui keytool generate ed25519  # network key
```

Each command outputs a keypair file. Store the key files securely. The authority key (BLS12-381) is the most sensitive — it is used to sign consensus votes.

:::danger Protect your authority key

The authority key has the highest security requirements. If compromised, an attacker can sign malicious consensus votes on behalf of your validator. Consider generating it on an air-gapped machine (see Step 3).

:::

## Step 2: Generate Proof of Possession

A Proof of Possession (PoP) proves that you control the authority key. It prevents rogue-key attacks where an attacker registers a public key they do not own.

Generate the PoP alongside your validator metadata:

```bash
sui validator make-validator-info   --name "My Validator"   --description "Description of my validator"   --image-url "https://example.com/logo.png"   --project-url "https://example.com"   --host-name "validator.example.com"   --gas-price 1000
```

This command prompts you to select the authority key and generates the PoP automatically. The output includes a `validator.info` file containing your validator metadata and PoP, which you submit when registering as a validator candidate.

## Step 3: Cold wallet setup

The **account key** is the key that controls your validator's Sui address — the address that receives staking rewards, holds the `OperationCap`, and manages the validator's configuration. This key should never be on the validator node.

### Air-gapped key generation

1. On an air-gapped machine (no network connection), install the Sui CLI.
2. Generate an Ed25519 keypair:
   ```bash
   sui keytool generate ed25519
   ```
3. Record the Sui address and back up the keypair file to secure offline storage (encrypted USB, safe deposit box).
4. Transfer only the **address** (not the key) to your online machine for use in validator registration.

The account key is used infrequently — only for administrative operations like changing validator metadata, withdrawing rewards, or creating an Operation Cap. Keep it offline between uses.

## Step 4: Ledger hardware wallet

For validators who prefer hardware wallet security over air-gapped file management, use a Ledger device for the account key.

### Setup

1. Install the Sui app on your Ledger device through Ledger Live.
2. Use the `@mysten/ledger-signer` package to interact with the Ledger from your application or scripts:

```bash
npm install @mysten/ledger-signer
```

```typescript
import { LedgerSigner } from '@mysten/ledger-signer';

const signer = await LedgerSigner.connect();
const address = await signer.getAddress();
console.log('Ledger Sui address:', address);
```

3. Use this address as your validator's account address during registration.

### Signing transactions with Ledger

When you need to sign an administrative transaction (creating an Operation Cap, changing metadata), connect the Ledger and sign:

```typescript
import { Transaction } from '@mysten/sui/transactions';
import { SuiGrpcClient } from '@mysten/sui/grpc';

const client = new SuiGrpcClient({
  baseUrl: 'https://fullnode.mainnet.sui.io:443',
  network: 'mainnet',
});

const tx = new Transaction();
tx.setSender(await signer.getAddress());
// ... add transaction commands ...

const result = await client.signAndExecuteTransaction({
  transaction: tx,
  signer,
});
```

The Ledger displays the transaction details on its screen for manual confirmation before signing.

## Step 5: Operation Cap

An `OperationCap` delegates day-to-day validator operations to a hot wallet so you do not need your cold wallet (or Ledger) for routine tasks. The Operation Cap authorizes operations like updating gas price, reporting peers, and requesting commission changes.

### Create an Operation Cap

Using your account key (cold wallet or Ledger), create an `OperationCap` and transfer it to a hot wallet address:

```bash
sui client call   --package 0x3   --module sui_system   --function create_operation_cap   --gas-budget 100000000
```

Then transfer the resulting `OperationCap` object to the hot wallet address that runs on your validator node:

```bash
sui client transfer   --to <HOT_WALLET_ADDRESS>   --object-id <OPERATION_CAP_ID>   --gas-budget 100000000
```

### What the Operation Cap authorizes

With an Operation Cap, the hot wallet can:

- Update the validator's gas price
- Report other validators
- Request commission rate changes
- Update validator metadata

The Operation Cap **cannot**:

- Withdraw staking rewards (requires the account key)
- Unstake delegations (requires the account key)
- Change the validator's account address

This separation means that even if the hot wallet is compromised, the attacker cannot steal staking rewards or take control of the validator's account.

## Step 6: Register as a validator candidate

Submit your validator info and stake to become a candidate:

```bash
sui validator join-committee   --validator-info-path validator.info
```

You need a minimum stake to become a validator candidate. The exact amount is set by the protocol and can be queried from the `SuiSystemState` object. Once registered, your validator becomes active at the start of the next epoch after receiving sufficient stake.

## Step 7: Verify your setup

After registration, verify that your validator is correctly configured:

```bash
# Check validator status
sui client call   --package 0x3   --module sui_system   --function active_validator_addresses   --gas-budget 10000000
```

Also verify on a Sui explorer ([SuiVision](https://suivision.xyz/) or [Suiscan](https://suiscan.xyz/)):

1. Search for your validator address.
2. Confirm your validator metadata (name, description, gas price) is correct.
3. Verify the authority key matches the one you generated.

For ongoing operations, monitoring, and troubleshooting, see [Validator Tasks](/operators/validator/validator-tasks).
