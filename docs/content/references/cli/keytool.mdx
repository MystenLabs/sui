The Sui CLI `keytool` command provides several command-level access for the management and generation of addresses, as well as working with private keys, signatures, or zkLogin. For example, you can export a private key from the Sui Wallet and import it into the local Sui CLI wallet using the `sui keytool import [...]` command.

## Commands

```sh
$ sui keytool --help
```

## JSON output

Append the `--json` flag to commands to format responses in JSON instead of the more human friendly default Sui CLI output. This can be useful for extremely large datasets, for example, as those results can have a troublesome display on smaller screens. In these cases, the `--json` flag is useful. 

## Examples

The following examples demonstrate some of the most often used commands.

### Generate a new key pair and store it in a file

To generate a new key pair with the `ed25519` scheme, use the `sui keytool generate ed25519` command. For other schemes, see `sui keytool generate –help`. The key pair file is saved to the current directory with its filename being the address. The content of the file is a Base64 encoded string of 33-byte `flag || privkey`. 

```sh
$ sui keytool generate ed25519
```

### Show the key pair data from a file

Use `sui keytool show [filename]` to show the key pair data that is stored in a file. For example, the previous command generated a file named `0x5d8aa70f17d9343813d3ba6a59ecf5e8a23ffb487938e860999a722989eaef25.key`. 

```sh
$ sui keytool show 0x5d8aa70f17d9343813d3ba6a59ecf5e8a23ffb487938e860999a722989eaef25.key
```

### Export a private key

Use `sui keytool export` to export the private key for an address that exists in the Sui CLI keystore. Pass the address with the `--key-identity` flag. The `--key-identity` value can be an address or an alias, as long as the CLI identity parser recognizes it.

```sh
$ sui keytool export --key-identity 0x3047f142a84297a42a65fb0a8c7a716d9d1b0bd0413d6bfa5ddfec45df175235
```
```sh
╭────────────────────┬─────────────────────────────────────────────────────────────────────────────╮
│ exportedPrivateKey │  suiprivkey1q...                                                             │
│ key                │ ╭─────────────────┬───────────────────────────────────────────────────────╮ │
│                    │ │ alias           │  elated-diamond                                       │ │
│                    │ │ suiAddress      │  0x3047f142a84297a42a65fb0a8c7a716d9d1b0bd0413d6bf...  │ │
│                    │ │ publicBase64Key │  AHsXwcxaWNaNtCIIszwu7V2G6HO8aNM1598w/8y0zI5q         │ │
│                    │ │ keyScheme       │  ed25519                                              │ │
│                    │ │ flag            │  0                                                    │ │
│                    │ │ peerId          │  7b17c1cc5a58d68db42208b33c2eed5d86e873bc68d335e7...  │ │
│                    │ ╰─────────────────┴───────────────────────────────────────────────────────╯ │
╰────────────────────┴─────────────────────────────────────────────────────────────────────────────╯
```

The `exportedPrivateKey` field is the private key encoded in Bech32 format. It always starts with `suiprivkey1`. You can pass this value to `sui keytool import` to add the key to another Sui CLI keystore.

This command exports from the Sui CLI keystore at `~/.sui/sui_config/sui.keystore`. It does not export keys from browser or mobile wallets, unless you previously imported that key into the Sui CLI keystore.

:::danger

The exported private key grants full control of the account, including all of its assets. Anyone who obtains it can sign transactions as you, and the action is irreversible. Never paste a private key into a chat, commit it to a repository, write it to logs, store it in CI variables, or place it in a frontend `.env` file. Keep it only in a secure secret manager or keystore.

:::

To add an exported key to a keystore, use `sui keytool import`. To sign transactions without exposing the key to a connected environment, see [Offline signing](/develop/transactions/transaction-auth/offline-signing). For the keystore location and client setup, see [Configure a Sui Client](/getting-started/onboarding/configure-sui-client).
