---
title: Sui Replay CLI
description: Use the Sui CLI to replay a given transaction to view its effects.
---

Replay is a tool that allows you to replay a given transaction to check and analyze the transaction effects. In a nutshell, the replay tool will locally execute the transaction and compare its effects to the effects that exist on chain. 

With Replay, you can pass the `--trace` flag to generate execution traces that can be used with the Move debugger to debug that transaction locally. For more information about the Move debugger, see the [Move Debugger](../ide/debugger.mdx) page.

## Usage 
The main command for replaying a transaction follows. For all available flags and arguments, run `sui client replay-transaction --help`.

```
sui client replay-transaction --tx-digest <digest>
```

After running this command, the tool generates a directory corresponding to the digest in the current directory: `<cur_directory>/.replay/<digest>`, unless you set the `--output-dir` flag.
