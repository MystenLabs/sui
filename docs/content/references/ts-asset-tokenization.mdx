The asset tokenization setup scripts expose a set of commands for interacting with deployed tokenized assets. Run all commands from the `setup` directory. Commands are defined in [`setup/src/calls.ts`](https://github.com/MystenLabs/asset-tokenization/blob/main/setup/src/calls.ts) and implemented in [`setup/src/functions`](https://github.com/MystenLabs/asset-tokenization/tree/main/setup/src/functions).

Before running any commands, make sure you have [deployed the packages and populated your `.env` file](/guides/developer/digital-assets/non-fungible-tokens/deploy-tokenized-asset).

## `create-tp`

Creates a `TransferPolicy` and a `ProtectedTP` for the tokenized asset type. Store the `TransferPolicy ID` and `ProtectedTP ID` from the transaction output in your `.env` file.

```sh
$ npm run call create-tp
```

## `tp-rules`

Adds transfer rules to the existing `TransferPolicy`. Edit `transferPolicyRules.ts` to define which rules to apply. Available rules include floor price, lock, royalty, and personal kiosk. Rules are chainable:

```ts
tpTx
  .addFloorPriceRule('1000')
  .addLockRule()
  .addRoyaltyRule(percentageToBasisPoints(10), 0)
```

```sh
$ npm run call tp-rules
```

## `mint`

Mints a new tokenized asset and transfers it to the owner address. Edit the `getVecMapValues` function in `mint.ts` before running: providing key-value pairs mints an NFT with a balance of 1; leaving the arrays empty mints an FT with a configurable balance. Store the returned object ID in `TOKENIZED_ASSET` in your `.env` file.

```sh
$ npm run call mint
```

## `mint-lock`

Mints a new tokenized asset and immediately locks it in the kiosk in a single transaction. Equivalent to running `mint` then `lock` in sequence.

```sh
$ npm run call mint-lock
```

## `select-kiosk`

Selects the personal kiosk to use for this project. The setup requires a single kiosk per owner. Store the returned kiosk ID in `TARGET_KIOSK` in your `.env` file.

```sh
$ npm run call select-kiosk
```

## `lock`

Locks a tokenized asset in the kiosk, preventing usage outside the defined transfer policy. Populate `TOKENIZED_ASSET` in your `.env` file before running.

```sh
$ npm run call lock
```

## `list`

Lists a locked tokenized asset for sale in the kiosk. Edit the sale price in `listItem.ts` before running.

```sh
$ npm run call list
```

## `purchase`

Purchases a listed tokenized asset from a kiosk. Edit `purchaseItem.ts` to set the item ID, price, and seller kiosk ID before running.

```sh
$ npm run call purchase
```

## `join`

Merges 2 FT tokenized assets. The balance of `FT2` is added to `FT1`, and `FT2` is burned. Only FT type assets can be joined. Populate `FT1` and `FT2` in your `.env` file before running.

```sh
$ npm run call join
```

## `burn`

Permanently destroys a tokenized asset and decreases the circulating supply by its balance. Only works if the asset was created with `burnable` set to `true`. Populate `TOKENIZED_ASSET` in your `.env` file before running.

```sh
$ npm run call burn
```

## `get-balance`

Returns the balance of a specified tokenized asset. Populate `TOKENIZED_ASSET` in your `.env` file before running.

```sh
$ npm run call get-balance
```

## `get-supply`

Returns the current circulating supply of the asset.

```sh
$ npm run call get-supply
```

## `get-total-supply`

Returns the maximum total supply of the asset as defined at creation time.

```sh
$ npm run call get-total-supply
```
