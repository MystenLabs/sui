This cookbook shows how to combine zkLogin, Enoki, and Seal to build consumer applications where users interact with Sui without seeing seed phrases, gas fees, or wallet popups.

## Overview

Consumer applications targeting non-crypto users need to eliminate every point of friction in the onboarding flow. The Sui stack provides three primitives that, when combined, create an invisible-crypto experience:

| Primitive | What it handles |
| --- | --- |
| [zkLogin](/sui-stack/zklogin-integration) | Authentication — users sign in with Google, Apple, or other OAuth providers instead of managing keys |
| [Enoki](https://docs.enoki.mystenlabs.com/) | Managed infrastructure — handles salt storage, proof generation, and sponsored transactions behind a single API |
| [Seal](/sui-stack/seal) | Encrypted storage — lets you gate access to content or data based on onchain conditions without exposing it publicly |
