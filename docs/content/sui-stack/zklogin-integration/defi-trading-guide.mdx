This guide shows how to build a DeFi trading interface where users authenticate with zkLogin, execute swaps, and never manage keys or pay gas directly.

## Overview

DeFi applications on Sui can use zkLogin to onboard users who do not have a Sui wallet. Combined with sponsored transactions, this creates a trading experience where the user signs in with Google, swaps tokens, and the application covers gas fees — eliminating the cold-start problem of needing SUI before you can trade.

## Considerations

- **Session management:** Cache the ephemeral key pair and ZK proof in session storage. Users should not need to re-authenticate for every trade.
- **Gas budget:** The sponsor sets the gas budget. Monitor gas costs to avoid sponsoring transactions that exceed your budget.
- **Price slippage:** DeFi transactions can fail if prices move between building and execution. Set appropriate slippage tolerances.
- **Multi-coin support:** The zkLogin address starts with no tokens. Consider how users fund their address or integrate with fiat on-ramps.
