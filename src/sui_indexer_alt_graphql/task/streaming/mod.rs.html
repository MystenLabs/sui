4//! Streaming subscription pipeline.
5//!
6//! # 1. Initialization: filling the store while waiting for readiness
7//!
8//! gRPC starts streaming at checkpoint C. Packages from checkpoints < C live
9//! only in the DB, so subscriptions must wait until kv_packages has indexed
10//! them before being served.
11//!
12//! Meanwhile, checkpoints C, C+1, C+2, ... populate the store:
13//!
14//! ```text
15//!   gRPC stream:   C,   C+1,   C+2,   ...
16//!                  ↓     ↓      ↓
17//!   StreamingPkgStore  stores packages from each streamed checkpoint
18//!
19//!   kv_packages_hi:  [.......... must reach ≥ C-1 ..........]
20//!                                      │
21//!                                      ▼
22//!          Subscriptions unblock; start receiving from current tip (C+N)
23//! ```
24//!
25//! # 2. Eviction: draining the store as kv_packages catches up
26//!
27//! After startup, the stream keeps advancing while kv_packages indexes in the
28//! background. Packages in the store at checkpoints ≤ kv_packages_hi are safely
29//! in the DB and can be removed. Periodic eviction keeps the store bounded.
30//!
31//! ```text
32//!   gRPC stream:       ..., C+10, C+11, C+12, C+13, C+14, ...
33//!   kv_packages_hi:    .........  C+11 ............
34//!
35//!                               │
36//!                               ▼
37//!   StreamingPkgStore keeps packages at cp > C+11 (C+12, C+13, C+14, ...)
38//!   Packages at cp ≤ C+11 are evicted and served by:
39//!     PackageCache (shared, LRU + system-package invalidation) → DB
40//! ```
