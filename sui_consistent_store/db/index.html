The Db handle wrapping an opened RocksDB database.

Db is a cheap-to-clone handle: every clone is one Arc bump against a shared inner that holds the actual RocksDB database. Clones are shared freely across typed column-family handles and across threads; the database itself stays alive until the last clone drops, at which point RocksDB’s own shutdown sequence (flush plus close) runs.

Db also holds the in-memory snapshot buffer used to serve consistent reads at a given checkpoint. See take_snapshot, at_snapshot, and Snapshot.

RocksDB is internally thread-safe; the only external locking the crate adds is a [parking_lot::RwLock] over the snapshot buffer.
