Typed iteration over a column family.

Two iterator types are exposed: Iter for forward iteration in lexicographic key order, and RevIter for reverse iteration. Both implement Iterator yielding Result<(K, V), Error> so that decode failures are reported as per-item errors rather than poisoning the entire scan; the iterator stops yielding after the first error.

Construct an iterator via DbMap::iter, iter_rev, iter_prefix, or iter_rev_prefix.

§Range and prefix iteration

iter and iter_rev take a RangeBounds<K> and produce iterators bounded by it. Use .. for an unbounded scan, start..end (or any combination of Included / Excluded bounds) for a typed range.

iter_prefix and iter_rev_prefix accept a separate prefix type whose Encode form is treated as a byte prefix of the full key’s encoded form. The crate does not statically verify that the prefix encoding really is a prefix of the key encoding; that is a property of the schema’s chosen encoding scheme. In practice, schemas that encode compound keys with a prefix-preserving representation (for example, big-endian fixed-int tuples) and pass tuple prefixes get the right behavior.

Internally, both range and prefix iteration set RocksDB’s iterate_lower_bound and iterate_upper_bound to the corresponding byte sequences, so the underlying scan stops cleanly at the bound without per-item filtering on the Rust side. The same byte-level helper backs all four entry points.

§Cursor methods

Iter and RevIter expose lower-level cursor methods — seek, skip_past, raw_key, raw_value, and valid — for advanced traversal patterns (for example, byte-cursor pagination that does not want to round-trip through K’s decoder).
