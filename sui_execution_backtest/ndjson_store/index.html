Module ndjson_store 

Source
Expand description

A zero-setup ConcurrentStore that appends rows to a newline-delimited JSON file instead of a database. It lets the same Indexer/Handler pipeline run without a postgres dependency (for quick local smoke tests); the postgres path is the durable, queryable one.

Watermarks are kept in memory only, so an ndjson run does not resume across process restarts — acceptable for the bounded smoke runs this sink is for. Persisting them to a sidecar file (to match the postgres path’s resumability) is a deliberate non-goal for now.
