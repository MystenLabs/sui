Module event_bitmap_index 

Source
Expand description

Roaring-bitmap inverted index keyed by packed event sequence number.

Transactions have a global tx_seq, but events are only ordered within their containing transaction. We pack (tx_seq, event_idx) into one globally ordered u64:

event_seq = (tx_seq << EVENT_BITS) | event_idx

EVENT_BITS = 16 reserves 65,536 event slots per transaction. The protocol config’s max_num_event_emit is currently 1,024, so this gives 64x headroom for future protocol changes while leaving 48 bits for tx_seq (~281T historical transactions, over 55k times the chain’s current transaction count at the time this index was introduced). Even at a sustained 1M TPS, 48 tx bits provide roughly 8.9 years of transaction history before a schema bump would be needed.

The packed namespace is sparse: most transactions use only a tiny prefix of their reserved event slots. Each row spans 268,435,456 packed event positions (BUCKET_SIZE = 2^28), which is exactly 4,096 transactions worth of event namespace at EVENT_BITS = 16, while leaving bucket-relative bit positions (max 2^28 - 1) well within RoaringBitmap’s u32 limit.
