Watermarks table: stores per-pipeline watermarks indexed by pipeline name.

A row in the v1 schema has one cell per WatermarkV1 field (u64 big-endian), plus a schema-version tag cell and an optional BCS-encoded WatermarkV0 cell:

  • ehi / chi / th / tmhi / rl / ph / ptm / b: per-field u64 BE cells. chi (checkpoint_hi_inclusive) is absent when the committer has not observed a checkpoint yet (Option<u64>::None).
  • v: u64 BE schema version. Rows in the current schema carry v = SCHEMA_V1; create_pipeline_watermark_if_absent keys off this cell to detect row existence, and a future format migration can bump it to branch read behavior.
  • w (v0): BCS-encoded WatermarkV0, kept in sync alongside the per-field cells so existing BCS consumers keep working.
  • b (bitmap-only): replay-floor checkpoint for the currently-active bitmap bucket. init_watermark resumes from just before this checkpoint so the active bucket is rebuilt after a crash. 0 means no bucket transition has been observed yet. Absent on non-bitmap pipelines and older rows.

Functions§

decode_v0
Reads only the v0 w cell. Returns Ok(None) if absent. Used only by init_watermark to bootstrap the v1 format from a v0-only row.
decode_v1
Strict read of the v1 cells. Returns Ok(None) if the schema-version v cell is absent (v0-only row or row does not exist). Bails on unknown schema versions, and on missing required field cells when v is present (which would indicate data corruption since all required field cells are written atomically with v). chi and the bitmap-only b field are optional.
encode_committer_cells
Cells for a committer-watermark write: the v1 committer fields + the BCS v0 cell + an optional bitmap b cell. Used by BigTableClient::set_committer_watermark_cells. Reader/pruner columns are written separately by ConcurrentConnection.
