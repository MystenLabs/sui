Expand description

A Roaring-bitmap inverted index definition.

The generic handler owns the shared checkpoint-to-row accumulation logic. Individual processors only describe how a checkpoint contributes bits and how those bits map into table-specific row keys.

fn is_sealed(bucket_id: u64, watermark: CommitterWatermark) -> bool

Returns true when the persisted committer watermark guarantees no future checkpoint will contribute a bit to bucket_id. The committer uses this to advance the active bucket and evict sealed rows from in-memory shard state.

Each processor decides which watermark dimension matters: tx-keyed bitmaps gate on tx_hi, checkpoint-keyed bitmaps gate on checkpoint_hi_inclusive, etc.
